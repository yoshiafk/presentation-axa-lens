SecureGPT OCR in Action

Real-world scenarios demonstrating intelligent error correction and validation capabilities

Post-OCR NLP Correction

Scenario: OCR misreads passport number 'A1234567' as 'A1234S67'
Input: "A1234S67"
Analysis:

Context-aware character validation detects 'S' between digits

Validation:

Indonesian passport format: 1 letter + 7 digits

Correction:

Character similarity analysis (S vs 5, 6, 7)

Output: "A1234567" Confidence: 0.92

Fuzzy Airport Code Matching

Scenario: OCR extracts 'CQK' instead of 'CGK' (Soekarno-Hatta)
Input: "CQK"
Algorithm:

Fuzzy matching with Levenshtein distance calculation

Analysis:

CGK distance: 1 (Q→G substitution), common OCR error pattern

Validation:

Cross-reference with airport database

Output: "CGK" (Soekarno-Hatta International Airport) Confidence: 0.87

Context-Aware Field Extraction

Scenario: Multiple names in ticket text - extract correct passenger name
Input: "GARUDA INDONESIA\nJOHN DOE\nJAKARTA SINGAPORE\nCAPT SMITH"
Algorithm:

Named Entity Recognition with context analysis

Process:

Identify name patterns, analyze context clues (airline, destination, crew indicators)

Validation:

Apply passenger name formatting rules and ticket structure validation

Output: "JOHN DOE" Confidence: 0.94

MRZ Validation and Extraction

Scenario: Extract passport data from Machine Readable Zone
Input: P<IDNDOE<<JOHN<<<...A12345678IDN8501019M2501017
Algorithm:

ICAO MRZ parsing with checksum validation

Process:

Validate format (44 characters), extract fields using positional parsing, calculate checksums

Output:

Passport: A1234567 | Nationality: IDN

Name: JOHN DOE | DOB: 1985-01-01

Expiry: 2025-01-01 | Gender: Male

Confidence: 0.96

Ensemble Field Scoring

Scenario: Multiple extraction strategies provide different flight numbers
Strategies:

Pattern-based: "GA123" (0.88)

Keyword-based: "GA123" (0.76)

Positional: "GA133" (0.62)

Airline-specific: "GA123" (0.94)

Algorithm:

Weighted ensemble voting with confidence weighting

Calculation:

GA123: (0.88 + 0.76 + 0.94) / 3 = 0.86
GA133: 0.62 / 1 = 0.62

Output: "GA123" Ensemble Confidence: 0.86

Confidence-Based Fallback

Scenario: Low confidence flight number triggers reprocessing
Initial Result: "G4123" Confidence: 0.43
Detection:

Confidence below threshold (0.5) triggers fallback

Fallback Strategy:

Apply alternative extraction strategies, use positional and context-based extraction

Validation:

Validate against airline prefixes (GA for Garuda Indonesia)

Final Result: "GA123" Confidence: 0.89
